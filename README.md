# Bot_Mont_SHK

Telegram-бот принимает Excel-файл, агрегирует данные и загружает результат в Google Sheets. ШК остаются внутри одной ячейки, разделённые `\n`.

## Возможности
1. Принимает `.xlsx` файл в Telegram.
2. Предлагает режимы обработки:
   - Выгрузка без передач
   - Выгрузка с передачами
   - Выгрузка только передач
3. Агрегирует по колонке `Гофра`, суммирует `Стоимость`, считает количество ШК и оставляет все ШК в одной ячейке.
4. Фильтрует группы по порогу `Стоимость > 2000`.
5. Загружает результат в диапазон `D:G` указанного листа Google Sheets одним обновлением.

## Требования
- Python 3.10+
- Сервисный аккаунт Google (JSON-файл с ключом) с доступом к нужной таблице.

## Установка и запуск на Ubuntu
```bash
cd Bot_Mont_SHK
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
# заполните .env
python main.py
```

Запуск возможен и с другим путём к `.env`, задав `BOT_CONFIG_FILE=/path/to/.env`.

## Переменные окружения
- `TELEGRAM_TOKEN` — токен бота (можно также `TELEGRAM_BOT_TOKEN` для совместимости).
- `SPREADSHEET_ID` — ID Google Sheets.
- `WORKSHEET_NAME` — имя листа (опционально, по умолчанию `sheet1`).
- `GOOGLE_CREDENTIALS_PATH` — путь к JSON сервисного аккаунта (проверяется при старте).
- `ADMIN_USER_ID` — Telegram ID администратора (опционально, нужен для /admin).
- `BOT_CONFIG_FILE` — путь к кастомному `.env` (опционально).

## Как пользоваться
1. Отправьте боту Excel-файл `.xlsx` (до 20 МБ).
2. Выберите тип выгрузки в инлайн-кнопках.
3. Бот последовательно сообщает этапы: чтение → обработка → загрузка → итог.
4. В результате получаете ссылку на таблицу и краткую сводку.

## Админка
- Команда `/admin` доступна только `ADMIN_USER_ID`.
- Действия:
  - Просмотреть логи (последние ~200 строк; при большом объёме — как документ).
  - Запустить новую задачу (подсказка загрузить файл).
  - Остановить бота.

## Логи
- Пишутся в `logs/bot.log` и stdout.
- Формат: время, уровень, модуль, сообщение; используется ротация (5 МБ, 3 бэкапа).

## Примечания
- ШК всегда хранятся в одной ячейке, разделённые `\n`.
- Одна обработка за раз (используется асинхронный lock).
- Диапазон очистки ограничен столбцами `D:G` и фактическим количеством строк, чтобы не трогать остальные данные листа.
