# Bot_Mont_SHK

Telegram-бот для обработки Excel-файла и загрузки агрегированных данных в Google Sheets.

## Что делает бот

1. Принимает `.xlsx` файл в Telegram.
2. Дает выбор режима обработки:
   - `Выгрузка без передач`
   - `Выгрузка с передачами`
   - `Выгрузка только передач`
3. Агрегирует данные по колонке `Гофра`.
4. Фильтрует группы по порогу `Стоимость > 2000`.
5. Загружает результат в Google Sheets (диапазон `D:H`).

Передачи: `Гофра` начинается с `3` или `4`.  
Тара: `Гофра` начинается с `7`, `9` или `10`.  
Нестандартные значения (например `0`) также обрабатываются с тем же порогом по стоимости.

## Требования к входному файлу

Обязательные колонки:
- `Гофра`
- `ШК`
- `Стоимость`

## Настройка окружения

Создайте файл `Bot_Mont_SHK/.env` (можно скопировать из `.env.example`) и заполните:

```env
TELEGRAM_BOT_TOKEN=...
GOOGLE_CREDENTIALS_PATH=...
SPREADSHEET_ID=...
ADMIN_USER_ID=...
```

`BOT_CONFIG_FILE` задавать не обязательно.  
Если не задан, бот читает `Bot_Mont_SHK/.env`.

## Локальный запуск (Windows)

```powershell
.\.venv\Scripts\python.exe Bot_Mont_SHK\Bot_Telegram_SHK_bez_dv_1.1.py
```

Важно: запускайте только один экземпляр бота, иначе будет ошибка `409 Conflict`.

## Запуск на Yandex VM (Linux)

1. Установите Python и зависимости.
2. Передайте переменные окружения или используйте `.env`.
3. Запускайте один процесс бота.

Пример ручного запуска:

```bash
python3 Bot_Telegram_SHK_bez_dv_1.1.py
```

## Безопасность (уже учтено в коде)

- Секреты читаются из env, не из кода.
- Токен маскируется в логах.
- Ограничен размер входного файла.
- Проверяются обязательные колонки.
- Одна выгрузка за раз (lock от гонок).
- Временные файлы удаляются после обработки.
