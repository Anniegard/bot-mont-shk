# Bot_Mont_SHK

Telegram-бот принимает два типа файлов и пишет результат на один лист Google Sheets:  
- «Без движения» (основная выгрузка)  
- «24 часа» (прогноз списаний)  

Данные выводятся в две зоны листа с одной пустой строкой между записями для удобного `CTRL+A`.

## Возможности
- Постоянные кнопки:  
  - 📦 Без движения — загрузка основного Excel.  
  - ⏱ 24 часа (обновить) — загрузка прогноза, обновление snapshot.  
  - ☁️ Взять с Я.Диска (последний файл) — скачать последний файл из указанной папки Я.Диска по OAuth.  
  - 📎 Инструкция по загрузке на Диск.  
  - 🛠 Админ-панель (только для ADMIN_USER_ID).
- Приём файлов: документ в Telegram до 20 МБ, прямая ссылка, Яндекс.Диск (OAuth, без публичных ссылок), zip с Excel внутри.
- «Без движения»: группировка по «Гофра», объединение идентификаторов через `\n`, фильтр `Стоимость > 2000`, колонка «Идентификатор товара».
- «24 часа»: чтение snapshot, пересечение с картой ID тары из «Без движения», группировка по ID тары, MIN прогноза, сортировка по времени.
- Google Sheets: две зоны на одном листе, заголовки и метка актуальности snapshot 24ч.
- Логирование действий в `logs/bot.log` (ротация 5MB×3) + stdout.

## Структура листа Google Sheets
- Левая зона («Без движения»):  
  - B2:E2 — «Выгрузка Идентификатор товара без движения» (объединено)  
  - B3:E3 — колонки: `Гофра`, `Идентификатор товара`, `Кол-во`, `Стоимость`  
  - Данные с B5, без пустых строк между записями.
- Правая зона («24 часа»):  
  - K2:O2 — «Товар, который спишется в течение 24ч» (объединено)  
  - K3:O3 — колонки: `ID тары`, `Идентификатор товара`, `Кол-во`, `Стоимость`, `Когда начнёт списываться?`  
  - Данные с K5, без пустых строк между записями.
- P3 — «Актуальность файла 24ч:»; P4 — время последнего snapshot.

## Установка и запуск (Ubuntu)
```bash
cd Bot_Mont_SHK
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env   # заполните переменные
python main.py
```

## Переменные окружения
- `TELEGRAM_TOKEN` или `TELEGRAM_BOT_TOKEN` — токен бота.
- `SPREADSHEET_ID` — ID таблицы.
- `WORKSHEET_NAME` — имя листа (опционально, по умолчанию первый).
- `GOOGLE_CREDENTIALS_PATH` — путь к JSON сервисного аккаунта.
- `ADMIN_USER_ID` — Telegram ID администратора (опционально).
- `BOT_CONFIG_FILE` — путь к .env, если используется другой файл.
- Яндекс.Диск (OAuth):  
  - `YANDEX_OAUTH_TOKEN` — OAuth-токен Диска.  
  - `YANDEX_NO_MOVE_DIR` (рекомендуется `disk:/BOT_UPLOADS/no_move/`).  
  - `YANDEX_24H_DIR` (рекомендуется `disk:/BOT_UPLOADS/24h/`).  
  - `YANDEX_ALLOWED_EXTS` — по умолчанию `.xlsx,.xls,.zip`.  
  - `YANDEX_MAX_MB` — лимит скачивания по OAuth (по умолчанию 200).

## Файлы данных
- `data/block_ids.txt` — whitelist ID Блока для фильтра 24ч (по одному в строке). Шаблон: `data/block_ids.txt.example`.
- `data/last_24h_snapshot.json`, `data/last_24h_meta.json` — сохраняются после загрузки 24ч.
- `data/last_no_move_map.json`, `data/last_no_move_meta.json` — карта идентификатор → ID тары из «Без движения» (используется как fallback для правого блока).

## Использование
1. Отправьте /start — появится клавиатура.  
2. Выберите режим:  
   - 📦 Без движения — пришлите Excel (до 20 МБ) или ссылку, либо возьмите последний файл с Диска.  
   - ⏱ 24 часа — пришлите прогноз или нажмите «☁️ Взять с Я.Диска…»; после загрузки правый блок обновляется сразу (если есть карта ID тары).  
3. Кнопка «📎 Инструкция по загрузке на Диск» — напоминание, в какие папки класть файлы.  
4. Админ-панель: только `ADMIN_USER_ID`, можно просмотреть логи или остановить бота.

## Яндекс.Диск (OAuth, без публичных ссылок)
- Создайте папки: `disk:/BOT_UPLOADS/no_move/` и `disk:/BOT_UPLOADS/24h/`.  
- Убедитесь, что токен имеет доступ к этим папкам.  
- Для загрузки файла: положите его в соответствующую папку; бот возьмёт самый свежий Excel/zip по кнопке «☁️ Взять с Я.Диска…».

## Логи
- `logs/bot.log` + stdout, формат: время, уровень, модуль, сообщение.  
- Логируются реальные действия: выбор режима, загрузка/источник файла, количество строк/групп, ошибки.

## Примечания
- Пустые/невалидные прогнозы 24ч исключаются; группы без корректного прогноза не выводятся.  
- Фильтр 24ч по `ID Блока` использует `data/block_ids.txt`; пустой файл = без фильтра.  
- Одна обработка за раз (async lock).  
- Документы через Telegram — до 20 МБ; большие файлы отправляйте ссылкой или через Я.Диск.
